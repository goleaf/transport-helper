# Root Cause Analysis

Root cause fields:

- category;
- summary;
- prevention notes;
- corrective-action required flag;
- explicit no-action reason.

Categories:

- `data_quality`
- `supplier_delay`
- `supplier_mismatch`
- `internal_process`
- `user_error`
- `integration_failure`
- `system_bug`
- `capacity_planning`
- `procurement_control`
- `master_data`
- `unknown`
- `other`

Critical and high incidents cannot be closed unless root cause is documented and either a corrective action exists or an explicit no-action reason is provided.
